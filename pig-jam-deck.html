02 · why now
Two trends point straight at agentic search.
query anatomy
3× longer
What length does to meaning
A long query collapses into one pooled vector and dilutes. Decompose, late-interaction, or rerank to fix it, and on an oblivious engine each is another full pass over the index. Length is not free.
the click is dying
56% → 69%
The answer replaces the click
Zero-click search rose 56 to 69% in a year. Agent traffic now exceeds human web traffic. Value is moving from the pageview to the synthesized answer.
02 · query anatomy
What length does to meaning
Conceptual, not measured: a single query embedding loses precision as the query lengthens; decomposition and late interaction hold the line.

A long query collapses into one pooled vector: every constraint averaged into a single point that drifts toward the center of the corpus. The more an agent specifies, the less it discriminates.

The fixes trade compute for fidelity: decompose, keep token-level vectors with late interaction (ColBERT MaxSim), or over-retrieve and rerank. Each multiplies the work per query.

On an oblivious engine this bites harder: every sub-query and every extra vector is another full linear pass over the index. Length is not free.

Or the agent fires many short queries instead. Then it is not dilution, it is volume. Either way the private engine pays, just on a different axis.

02 · the open problem
The research question.
Can an oblivious engine answer agent-grade queries, decomposed or multi-vector, without the cost collapsing to privacy’s linear scan times passes per query?
cost ≈ N × P N  index size; privacy forces a full scan
P  passes / query; 1 → k (decompose) → m (multi-vector)
01Pin the cost
One oblivious matvec is O(N) per query. Decomposition multiplies by k, late interaction by m. Is the true cost N·k, N·m, or can structure beat the product?
02Batched PIR
One agent query’s sub-queries are correlated: one client, one epoch, one intent. Can a batch share a single linear pass instead of k independent ones?
03Oblivious late interaction
ColBERT scores by MaxSim, an argmax over the index. Can MaxSim run under LWE without revealing which tokens or documents matched, below N·m cost?
04Leak-free decomposition
Splitting a query leaks its structure, the count, order, and routing, even when each part is individually private.
